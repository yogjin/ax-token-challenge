## 뭘 했는가

매일 대화 내용을 블로그 글로 옮기는 게 번거로워서, `/blog` 스킬을 만들었다. 대화 끝에 `/blog`을 입력하면 초안이 생성되고, 확인 후 파일이 저장된다.

### 해결한 문제

**대화 → 블로그 글 변환 자동화** — `~/.claude/skills/blog/SKILL.md` 하나로, 대화 내용을 ax-token-challenge 블로그 포스트로 변환하는 워크플로우를 만들었다.

## 어떻게 했는가

1. MCP 서버를 만들까 고민했지만, 블로그 글 생성은 Claude Code가 이미 할 수 있는 작업이라 스킬로 결정
2. `~/.claude/skills/blog/SKILL.md`에 절차를 정의: 슬러그 결정 → 대화 분석 → 초안 보여주기 → 확인 후 파일 생성
3. 전역 스킬이라 어떤 프로젝트에서든 `/blog`으로 호출 가능

## 깨달음

- **Claude Code가 이미 할 수 있는 일은 스킬로 지시만 정의하면 된다.** MCP 서버를 만들 필요가 없었다.
- **공식 문서 확인은 습관이다.** 어제 커맨드/스킬 통합을 잡아냈고, 오늘 바로 적용했다.
